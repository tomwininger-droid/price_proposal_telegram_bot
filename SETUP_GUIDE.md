# מדריך הקמה — בוט הצעות מחיר בטלגרם (ללא צורך בידע טכני)

מדריך הזה מסביר בדיוק איך להשיג כל פרט שתידרשו להדביק בטופס ההקמה, בסדר שבו הוא יופיע.
כל שורה כאן מתאימה לשדה אחד בטופס. אין צורך בידע ב-SSH, שרתים, או תכנות.

> **חשוב לדעת מראש**: הקמה אמיתית עדיין מחייבת אתכם ליצור חשבונות בכמה שירותים (טלגרם,
> OpenAI, Anthropic, ו-Render עצמו) ולהעתיק מהם ערכים. שום מדריך לא יכול לעשות את זה
> במקומכם — זו פעולה אנושית בכל פלטפורמה. מה שהמדריך הזה **כן** עושה: לוודא שכל שלב כזה
> לוקח כמה דקות, לא יותר, ושלא תזדקקו לכלום מעבר להעתק-הדבק.

## לפני שמתחילים: עלות
ההרצה התמידית של הבוט (כדי שיהיה זמין 24/7 ויזכור שיחות) עולה כ-$7-10 לחודש ב-Render
(או בכל פלטפורמה דומה). זה בנוסף לעלות שימוש בפועל ב-API של OpenAI/Anthropic (לפי כמה
הצעות הבוט מייצר בחודש — בדרך כלל סכומים קטנים מאוד).

לחצו "Use this template" (או קיבלתם קישור) ל-GitHub, וקבלו עותק משלכם של הקוד.

## שלב 2 — איסוף הפרטים (לפי הסדר שיופיע בטופס)

### TELEGRAM_BOT_TOKEN — חובה
1. בטלגרם, חפשו `@BotFather` ופתחו שיחה.
2. שלחו `/newbot`.
3. תנו שם תצוגה (לדוגמה: "ההצעות של הסטודיו שלי").
4. תנו שם משתמש שמסתיים ב-`bot` (לדוגמה: `MyStudioQuotesBot`).
5. תקבלו הודעה עם טוקן שנראה כך: `123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw`. העתיקו אותו.

### OPENAI_API_KEY — חובה
1. גלשו ל-[platform.openai.com](https://platform.openai.com), הירשמו/התחברו.
2. הגדירו אמצעי תשלום (Settings → Billing) — בלי זה המפתח לא יעבוד.
3. API Keys → Create new secret key. הוא מתחיל ב-`sk-`. **מוצג רק פעם אחת** — העתיקו עכשיו.

### ANTHROPIC_API_KEY — חובה
1. גלשו ל-[console.anthropic.com](https://console.anthropic.com), הירשמו/התחברו.
2. הגדירו אמצעי תשלום.
3. API Keys → Create Key. מתחיל ב-`sk-ant-`. העתיקו.

### ALLOWED_USER_IDS — חובה (אל תדלגו!)
1. בטלגרם, חפשו `@userinfobot` ושלחו לו כל הודעה.
2. הוא יחזיר `Id: 123456789` — העתיקו את המספר.
3. ⚠️ אם תשאירו את השדה הזה ריק, **כל אחד** שמוצא את הבוט שלכם יוכל להשתמש בו ולבזבז
   את הקרדיט שלכם ב-OpenAI/Anthropic. תמיד מלאו את ה-id שלכם (אפשר כמה, מופרדים בפסיק).

### מיתוג (BUSINESS_NAME / BUSINESS_SLOGAN / SENDER_NAME / PRIMARY_COLOR / BG_COLOR)
פשוט הקלידו את שם העסק שלכם, סלוגן קצר, ושם פרטי לחתימת מיילים (אופציונלי).
לצבעים — קוד hex (אם אין לכם, השאירו את ברירת המחדל הסגולה-אינדיגו, או שלחו לעיצוב/AI
לקבל קוד hex לפי הצבעים של המותג שלכם).

### SMTP_USER / SMTP_PASS / FROM_EMAIL — אופציונלי (שליחת מייל ללקוח)
דלגו על השלב הזה אם אתם לא צריכים שהבוט ישלח מייל ישירות ללקוחות (אפשר תמיד להוריד PDF
ולשלוח ידנית). אם כן רוצים:
1. ודאו שיש אימות דו-שלבי (2FA) מופעל בחשבון ה-Gmail/Google Workspace שלכם.
2. גלשו ל-[myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords).
3. צרו App Password (שם לדוגמה: "quote-bot"). תקבלו קוד של 16 תווים — זה ה-`SMTP_PASS`
   (לא הסיסמה הרגילה לחשבון!).
4. `SMTP_USER` ו-`FROM_EMAIL` הם כתובת המייל המלאה שלכם.

### DRIVE_FOLDER_ID / GOOGLE_SERVICE_ACCOUNT_KEY_B64 — אופציונלי (גיבוי הצעות בדרייב)
דלגו על השלב הזה אם אתם לא צריכים שכל הצעה תעלה אוטומטית לתיקיית Drive משותפת.
זה **השלב היחיד שמצריך רגע אחד בטרמינל** — אם זה מרתיע, אפשר לדלג, או לבקש ממישהו
שעשה את זה כבר לעשות לכם את הצעד הקטן הזה.

1. גלשו ל-[console.cloud.google.com](https://console.cloud.google.com), צרו פרויקט.
2. APIs & Services → Library → חפשו "Google Drive API" → Enable.
3. IAM & Admin → Service Accounts → Create Service Account (שם: למשל `quote-bot-drive`).
4. בתוך ה-Service Account → Keys → Add Key → Create new key → JSON. קובץ יורד למחשב.
5. פתחו את הקובץ, מצאו את `"client_email"` (נראה כמו `quote-bot-drive@...iam.gserviceaccount.com`).
6. בדרייב, פתחו את התיקייה שאתם רוצים → Share → הדביקו את ה-client_email → הרשאת Editor.
7. קודדו את כל קובץ ה-JSON לשורה אחת (זה הצעד הטכני היחיד):
   - **Mac**: פתחו Terminal, הריצו `base64 -i ~/Downloads/<קובץ>.json | pbcopy` — התוצאה
     הועתקה ללוח (Clipboard), פשוט הדביקו בטופס.
   - **Windows**: `certutil -encode <קובץ>.json key.b64` ואז פתחו את `key.b64` בעורך טקסט
     והעתיקו את התוכן (בלי השורות הראשונה/אחרונה שמתחילות ב-`-----`).
8. מזהה התיקייה (`DRIVE_FOLDER_ID`) הוא החלק שבכתובת ה-URL של התיקייה בדרייב, אחרי
   `/folders/` — לדוגמה מ-`drive.google.com/drive/folders/1AbCdEfGhIjK` המזהה הוא `1AbCdEfGhIjK`.

## שלב 3 — דיפלוי
1. בריפו שלכם ב-GitHub, לחצו על כפתור "Deploy to Render".
2. צרו/התחברו לחשבון Render.
3. ימולא טופס עם שדה אחד לכל ערך שאספתם בשלב 2 — הדביקו, לחצו Deploy.
4. בנייה ראשונה אורכת כמה דקות (Render מתקין את כל מה שצריך, כולל Chromium ל-PDF).

## שלב 4 — בדיקה
1. בדשבורד Render, מצאו את כתובת השירות (`https://<שם>.onrender.com`).
2. גלשו ל-`<הכתובת>/quote-tool/setup-status?token=...` — אם לא הגדרתם `SETUP_STATUS_TOKEN`
   בעצמכם, תצטרכו לבקש ממי שהקים את התבנית את הטוקן הנגזר, או להגדיר אחד ידנית בטופס.
3. ודאו שכל השורות החובה (Telegram, OpenAI, Anthropic) מסומנות ✅.
4. פתחו את הבוט שלכם בטלגרם (השם שבחרתם ב-BotFather), שלחו `/start`, ונסו הצעה ראשונה.

זהו — הבוט שלכם פעיל, עם המיתוג שלכם, ועם הנתונים שלכם.
